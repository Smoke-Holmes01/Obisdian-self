## 2. 按教材顺序总结概念

### 9.1 项目过程度量

- 定义：在项目执行过程中，对进度、成本、质量、范围、风险、资源和过程活动进行量化收集、分析和反馈。
- 目的：判断项目是否按计划运行，发现偏差，支持纠正和改进。
- 关键理解：过程度量关注项目运行状态和趋势，不是只看最终结果。

**度量流程**

```text
确定目标 -> 定义指标 -> 收集数据 -> 分析结果 -> 反馈改进
```

**常见度量指标**

- 进度：任务完成率、里程碑达成率、迭代完成率。
- 成本：实际成本、成本偏差、成本绩效指数。
- 质量：缺陷数、缺陷密度、测试通过率、缺陷关闭率。
- 范围：需求变更数、变更率。
- 风险：高优先级风险数量、风险关闭情况。

**度量规则**

- 目标导向、可量化、可收集、可解释、成本适当、可行动。

### 9.2 数据收集

| 收集方式 | 适用内容 |
| --- | --- |
| 人工记录 | 工时、问题清单、风险状态 |
| 系统自动采集 | Git、CI、缺陷系统、项目管理工具数据 |
| 会议汇报 | 阻塞、协调事项、状态同步 |
| 问卷调查 | 用户满意度、培训效果 |
| 访谈 | 深层需求、真实反馈 |
| 日志分析 | 性能、错误、访问行为 |
| 文档检查 | 交付物完整性、流程合规 |

**数据质量要求**

- 真实、完整、一致、及时、可追溯。
- 易错点：指标口径不统一、手工填报延迟、惩罚性使用数据，都会导致数据失真。

### 9.3 可视化管理

**可视化作用**

- 把复杂数据转化为直观图形。
- 帮助团队发现趋势、瓶颈、偏差和风险。
- 让项目经理、团队成员和干系人基于同一事实沟通。

**常见方法**

| 方法 | 主要用途 |
| --- | --- |
| 甘特图 | 展示总体进度、阶段和依赖 |
| 看板 | 展示任务流转和阻塞 |
| 燃尽图 | 展示迭代剩余工作量变化 |
| 累计流图 | 展示工作流瓶颈 |
| 缺陷趋势图 | 展示缺陷新增、关闭和遗留趋势 |
| 仪表盘 | 汇总关键项目指标 |

### 9.4 数据分析

**阶段关注点**

- 早期：需求稳定性、可行性、关键风险。
- 中期：进度偏差、质量趋势、资源瓶颈。
- 后期：缺陷收敛、验收问题、上线风险。

**分析方法**

- 趋势分析：看指标随时间如何变化。
- 偏差分析：比较计划与实际。
- 对比分析：和历史项目、基线或目标比较。
- 根因分析：找出问题背后的原因。
- 帕累托分析：找出影响最大的少数问题。

### 9.5 优先级控制

- 目的：资源有限时决定先做什么。
- 应用于：需求、任务、缺陷、风险和变更。

**缺陷严重性与优先级**

| 概念 | 含义 | 例子 |
| --- | --- | --- |
| 严重性 | 缺陷对系统或业务的影响程度 | 数据丢失、系统崩溃 |
| 优先级 | 缺陷应被处理的紧急程度 | 影响明天演示的问题优先修 |

**易考点**

- 严重性高通常优先级高，但两者不是同一个概念。

### 9.6 变更控制

**变更控制流程**

```text
提出变更 -> 记录变更 -> 影响分析 -> 评审决策 -> 实施变更 -> 验证确认 -> 归档
```

**影响分析内容**

- 范围、进度、成本、质量、风险、资源、合同和文档。

**变更控制目标**

- 让所有变更可见、可评估、可批准、可追溯。
- 防止口头变更直接进入开发，造成范围失控。

### 9.7 合同履行控制

- 控制内容：交付物、进度、费用、验收、变更、知识产权、维护服务和违约责任。
- 关键理解：合同履行控制把项目执行和合同承诺对齐，减少验收与结算争议。

<a id="chapter-summary"></a>
## 3. 本章总结

本章强调项目执行中的“看见、判断、纠偏”。监督控制要依靠数据而不是感觉，依靠可视化让状态透明，依靠优先级和变更控制保护项目目标。考试中重点掌握过程度量、数据收集质量、可视化方法、缺陷优先级与严重性区别、变更控制流程。

<a id="key-points"></a>
## 4. 本章重点内容

- 过程度量关注项目运行状态和趋势。
- 指标规则：目标导向、可量化、可收集、可解释、成本适当、可行动。
- 数据质量：真实、完整、一致、及时、可追溯。
- 可视化工具：甘特图、看板、燃尽图、累计流图、缺陷趋势图、仪表盘。
- 严重性是影响程度，优先级是处理顺序。
- 变更控制流程：提出、记录、影响分析、评审决策、实施、验证、归档。
- 合同履行控制要关注范围、交付、费用、验收和变更。

<a id="exercises"></a>
## 5. 习题及答案+解析

1. 什么是项目过程度量？其方法有哪些？

   答案：项目过程度量是在项目执行过程中，对进度、成本、质量、范围、风险、资源和过程活动进行量化收集、分析和反馈，以判断项目是否按计划运行并支持改进。方法包括直接度量、间接度量、基准比较、趋势分析、偏差分析、挣值分析、缺陷分析和统计过程控制。

   解析：过程度量关注“过程状态和趋势”，不是只看最终交付结果。

2. 过程度量的制定规则有哪些？请根据项目制定一些切实可行的度量指标。

   答案：规则包括目标导向、可量化、可收集、可解释、成本适当和可行动。学校教学管理系统可设置需求变更率、迭代计划完成率、任务准时完成率、缺陷密度、严重缺陷数、缺陷平均修复时间、测试用例执行率和通过率、用户验收问题数、数据迁移成功率等指标，并每周在项目例会上分析趋势。

   解析：指标必须能指导决策。不能设太多难采集、无法解释或无法行动的指标。

3. 数据信息收集的方式有几种？举例说明。

   答案：方式包括人工记录、系统自动采集、会议汇报、问卷调查、访谈、文档检查、日志分析和现场观察。例如从 Jira 获取任务状态，从缺陷系统获取缺陷数据，从 Git 和 CI 获取提交与构建结果，从服务器日志获取接口错误，从教师问卷获取易用性反馈。

   解析：自动采集更客观，访谈和问卷适合获取用户体验和深层问题。

4. 数据收集的难点在哪里？

   答案：难点在于真实性、完整性、一致性、及时性和成本控制。成员可能因担心考核而美化数据；不同人对缺陷严重级别理解不同；手工填报容易遗漏和延迟；指标过多会增加负担。解决办法是统一指标口径，尽量自动采集，减少惩罚性使用，结合多指标分析趋势。

   解析：数据本身可能失真，数据解释也可能失真。监督控制要重视指标口径。

5. 数据可视化的作用是什么？

   答案：数据可视化把复杂分散的数据转化为直观图形，帮助团队快速理解项目状态、发现趋势和定位问题。燃尽图、甘特图、累计流图、缺陷趋势图、看板和仪表盘可展示进度、质量、瓶颈和风险，并提高团队和干系人的沟通效率。

   解析：可视化不是为了好看，而是为了让异常更容易被发现、行动更容易被推动。

6. 结合实际项目你会选择哪个可视化方法来显示项目进度信息？

   答案：学校教学管理系统可结合甘特图、看板和燃尽图。甘特图向管理层展示总体计划、阶段、里程碑和依赖；看板用于团队日常管理，展示任务流转和阻塞；燃尽图用于迭代内控制，显示剩余工作量是否按预期下降。

   解析：不同可视化方法服务不同对象和管理层次，组合使用效果更好。

7. 简述变更控制的流程。

   答案：流程包括提出变更、记录变更、影响分析、评审决策、实施变更、验证确认和归档。干系人提交变更请求，项目组登记日志，分析对范围、进度、成本、质量、风险、合同和资源的影响，再由项目经理、客户代表或变更控制委员会决定批准、拒绝、延期或补充信息。获批后更新需求、计划、设计、测试用例和文档，实施后测试验收并归档。

   解析：变更控制的核心是“所有变更可见、可评估、可批准、可追溯”。

8. 结合你的项目，你会采取哪些有效方法来预防变更？

   答案：可通过充分访谈教务处、教师、学生和管理员，使用原型演示关键流程，为每个需求写清业务规则和验收标准，建立需求优先级，需求评审后形成基线，后续变更必须评估影响，采用迭代交付让用户尽早反馈，并把非紧急需求放入下一期。

   解析：预防变更不是禁止变化，而是减少因理解偏差和低价值需求造成的不必要变更。
